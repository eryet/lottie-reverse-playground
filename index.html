Workflow

How the implementation loop works in this repo.

The GitHub Pages showcase is visual by design, but the implementation method matters too. These are the three phases the current work follows.

01

Start from a motion brief

Begin with a high-level motion specification such as “a coin drops, rotates several times, then settles,” and let the LLM produce an initial implementation pass.

02

Refine iteratively

Review the motion, identify timing or behavior issues, and iterate with the LLM until the rotation, landing, pacing, and continuity match the intended result.

03

Reuse the implementation

Once the canvas version is stable, preserve the reusable structure and, for experiments like the dartboard, export the motion back into Lottie JSON output.
